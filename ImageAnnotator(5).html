Help & Reference ✕
Getting Started
1
Add Images — click "Add Images" in the toolbar or drag & drop image files. Images appear in the thumbnail strip at the bottom.
2
Define Classes & Objects — click "Classes" to set up object categories and their property schemas. Then use the "+" button in the Object Library (left panel) to add objects.
3
Annotate — click "Annotate" to enter annotation mode (cursor turns crosshair). Click on the image to place a marker; a picker appears to choose which object. Or arm an object first by clicking "Place on Image" in the library.
4
Export — use "Export" to download a full JSON database or a CSV of all sightings. Use "Import DB" to restore a saved session.

Left Panel — Object Library
List / Table toggleSwitch between grouped list view and a sortable, filterable table. In table view, columns match the class schema plus any properties selected in "Show Properties".
Search boxType a query and press Enter to filter. Use + to require multiple terms, e.g. red+1:72.
Class chips (bottom)Click a class chip to restrict the list to that class only.
Object row clickOpens the inline detail panel at the bottom of the left panel for quick viewing and editing.
Place on ImageArms the object and activates Annotate mode. The next click on the image places a marker for that object.
Right Panel — Sightings
Image PropertiesPer-image metadata fields defined in the Classes modal under "Image Fields". Values are saved per image — expand the header to edit them for the current image.
Show Properties barClick to expand. Toggle property chips to show those values on annotation labels, in the sightings list, and as table columns. Colorize changes dot/label colour by a chosen property value.
Sighting rowClick to select and highlight the annotation on the image. Use the edit icon to add a note; use the trash icon to delete.
Annotation Labels
Label contentShows the sighting note (if any) or the object name, plus any properties toggled on in the "Show Properties" bar.
LayoutLabels live in screen space and are automatically spread apart. They stay within the viewer panel and connect to their dot with a dashed leader line. Labels for off-screen dots are hidden.
Long textLabels wrap at approximately 28 characters per line.
Classes & Schemas
Object classesGroup objects by type (e.g. "Vehicle", "Person"). Each class can define a property schema — named fields with types (text, number, boolean, date, url) that appear automatically on every object of that class.
Image fieldsDefined separately from object schemas. These are per-image metadata fields (e.g. "Location", "Date") that appear in the collapsible "Image Properties" section of the right panel.
CSV ImportImports rows as objects into the library. Column name becomes the object name; color sets the marker colour; all other columns become properties.
Filters
Multi-term searchSeparate terms with + to require all terms to match, e.g. blue+sedan. Works in the object search box, the image filter field, and table column filters.
Image filterSelect a field from the dropdown, type a value, and press Enter. Only images matching that field value are shown in the thumbnail strip.
Viz filterIn table view, click "Viz" to apply the current column filters to the annotation layer — only matching objects are shown as markers.
Saved filtersIn table view, click "Save" to name and store the current column filter set. Saved filters appear as pills for quick recall.
Data & Storage
Auto-saveAll changes are automatically saved to the browser's localStorage. Data persists across page refreshes on the same browser/device.
Export JSONDownloads the full database (images, objects, classes, sightings, schemas) as a JSON file for backup or transfer.
Import DBLoads a previously exported JSON file. Choose "Merge" to add to existing data (duplicate IDs are skipped) or "Replace" to overwrite everything.
Image dataImages are stored as base64 data URIs in localStorage. Large images may approach browser storage limits; export regularly as a backup.
